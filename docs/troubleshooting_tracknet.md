# TrackNet 羽毛球检测失效问题排查与解决报告

**日期**: 2026-03-08  
**状态**: 已解决  
**模块**: `src/ai_engine/tracknet.py`

## 1. 问题描述

在项目初期测试中，TrackNet 模型在 CPU 上运行时表现异常，主要体现在：
1.  **无法检测到球**：在绝大多数帧中，模型输出的置信度（Confidence）为 `0.0000`。
2.  **坐标死锁**：在极少数有置信度（如 `0.9`）的帧中，输出坐标被死锁在固定位置（如 `(9, 3)`），不随球的实际移动而变化。
3.  **截图验证失败**：生成的 Debug 截图中，即便在明显有球的画面里，模型也无法标注出红点。

## 2. 排查过程

### 2.1 初步假设与验证
-   **假设 A：权重加载失败**
    -   *验证*：编写 `verify_weights.py` 脚本，确认 `.pt` 权重文件结构完整，且能正确加载到模型中。
    -   *结论*：排除。权重加载正常。
-   **假设 B：输入分辨率过低**
    -   *尝试*：将输入分辨率从 `288x512` 提升至 `640x360`。
    -   *结果*：问题依旧，且导致了严重的“坐标死锁”现象。这表明模型对输入尺寸非常敏感，不能随意更改。
    -   *结论*：排除。随意修改分辨率反而引入了新问题。

### 2.2 核心问题定位
通过分析 `debug_frames.csv` 和代码逻辑，我们锁定了两个关键因子：

1.  **ImageNet 归一化的副作用**
    -   *发现*：原代码中使用了 ImageNet 标准归一化（减去均值 `[0.485, ...]` 除以方差）。
    -   *分析*：TrackNet 的预训练权重是在仅做 `/255.0` 缩放的图像上训练的。引入 ImageNet 归一化破坏了图像的原始像素分布，导致特征图（Feature Map）数值偏移，激活函数失效。
    
2.  **分辨率不匹配**
    -   *发现*：TrackNet V1 的网络结构（3层池化）要求输入宽高必须是 8 的倍数，且预训练权重是针对 `288x512` 训练的。
    -   *分析*：使用非标准分辨率（如 `640x360` 或 `960x540`）会导致卷积核感受野与训练时不一致，甚至在反卷积（Upsampling）阶段产生维度对齐错误。

## 3. 解决方案

### 3.1 代码修正
在 `src/ai_engine/tracknet.py` 中进行了以下关键修改：

1.  **恢复标准分辨率**：
    强制将输入图像 resize 为 `(512, 288)` (W, H)。
    ```python
    resized = cv2.resize(frame_rgb, (512, 288))
    ```

2.  **修正预处理逻辑**：
    **移除** ImageNet 均值方差归一化，仅保留 `/255.0` 缩放。
    ```python
    processed = resized.astype(np.float32) / 255.0
    # 注释掉以下代码：
    # mean = np.array([0.485, 0.456, 0.406], ...)
    # processed = (processed - mean) / std
    ```

### 3.2 逻辑优化 (RallyAnalyzer)
针对检测结果的后处理，我们也解决了“回合重叠”的问题：
-   **现象**：由于对回合前后进行了时间扩展（向前0.6s，向后1.0s），导致原本临近的两个回合在时间轴上发生重叠。
-   **修复**：在 `src/decision/rally_analyzer.py` 中增加了 `_merge_overlapping_rallies` 方法，在输出前对重叠的回合进行贪婪合并。

## 4. 验证结果

使用 `test_video2.mp4` (Frame 300-4470) 进行验证：

-   **检测率**：模型能够稳定输出高置信度（`>0.8`）的检测框。
-   **轨迹连续性**：`debug_frames.csv` 显示球的坐标（x, y）呈现连续平滑的抛物线运动，符合物理规律。
-   **剪辑效果**：生成的 `highlights.mp4` 准确包含了完整的羽毛球回合，且没有画面回跳或重叠现象。

## 5. 经验总结

1.  **深度学习模型对预处理极其敏感**：在使用开源模型时，必须严格遵守其训练时的预处理步骤（分辨率、归一化方式、颜色空间），任何微小的差异都可能导致模型“致盲”。
2.  **调试工具的重要性**：`debug_frames.csv` 和可视化截图（Debug Screenshots）是定位计算机视觉问题的核心手段。单纯看日志无法发现坐标死锁等细微问题。
